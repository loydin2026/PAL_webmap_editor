# 仙剑奇侠传一代 地图编辑器（网页版）

基于原 Win32 SDK 版本重写的纯前端网页版地图编辑器，无需安装，浏览器直接运行。

## 快速开始

### 1. 启动本地服务器

由于浏览器安全策略限制，本地资源文件需要通过 HTTP 协议加载。

#### 方式一：使用内置 Node.js 服务器（推荐）

在 `web-map-editor` 目录中，双击运行：

```
start-server.bat
```

或者命令行：

```bash
cd web-map-editor
node server.js
```

然后打开浏览器访问：**http://localhost:8080/**

#### 方式二：使用 VS Code Live Server 插件

在 VS Code 中安装 "Live Server" 插件，右键 `index.html` → "Open with Live Server"。

#### 方式三：使用 Python（如已安装）

```bash
cd web-map-editor
python -m http.server 8080
```

### 2. 自动加载资源

页面打开后会**自动扫描并加载**资源：

1. **Pat.mkf**（调色板文件）—— 自动加载
2. **所有 GOP 图组文件**（gop0000 ~ gop0226）—— 扫描为列表，按需加载缓存
3. **所有 MAP 地图文件**（map0000 ~ map0226）—— 扫描为列表，按需加载缓存

页面加载完成后，默认会自动打开第一个可用的地图和对应的图组。

### 3. 下拉列表操作

- **左下角「地图」下拉列表**：选择不同的地图文件，加载对应的地图数据（位于地图编辑区右下角）
- **左侧「图块集」标签页**：选择不同的 GOP 图组文件，加载对应的图块到网格中
- **左侧「模板」标签页**：查看和管理自定义模板

当选中地图时，会自动加载其对应的 GOP 图组（编号匹配）。

### 4. 基本操作

| 操作 | 说明 |
|------|------|
| **左键点击** | 选择 Tile / 应用当前工具（画笔/擦除/障碍/模板/事件） |
| **左键拖拽** | 选择工具模式下拖拽平移视图；画笔模式下连续绘制 |
| **右键点击** | 选中 Tile 并显示属性 |
| **Shift + 点击** | 加选 Tile（多选） |
| **Ctrl + 点击** | 框选（拖动范围） |
| **Ctrl + Shift + 点击** | 减选 Tile |
| **Alt + 点击** | 减选框选（取消选中区域） |
| **滚轮** | 缩放（50% / 100% / 150% / 200% / 250% / 300%） |

### 5. 工具说明

- **🔍 选择**：拖拽平移地图，框选/多选/减选 Tile
- **✏️ 画笔**：在左侧面板选择图块后，点击地图绘制
- **📋 模板**：使用选中区域创建模板，点击地图粘贴
- **🧹 擦除**：清除当前图层的 Tile 数据
- **🚧 障碍**：切换 Tile 的障碍标记
- **🎬 事件**：在地图上放置/编辑事件对象（触发脚本、自动脚本等）
- **L0 / L1**：切换编辑的图层（第 0 层 / 第 1 层）
- **🚧 显示障碍**：切换障碍标记的显示/隐藏
- **🧍 显示人物**：在当前选中 Tile 上显示人物对象（用于遮挡关系预览）
- **🎬 显示事件**：切换事件标记在地图上的显示/隐藏

#### 工具与左侧面板的联动

- **画笔/模板/选择/擦除/障碍工具**：激活时显示左侧「图块集」和「模板」标签页，隐藏事件列表
- **事件工具**：激活时显示左侧「事件」列表，隐藏图块集和模板面板（事件编辑不需要图块）

### 6. 事件系统（SSS.MKF 支持）

#### 加载 SSS.MKF

点击左侧面板「事件」区域的 **📂 加载 SSS.MKF** 按钮，选择游戏的 `SSS.MKF` 文件：
- 自动解析事件数据和场景索引
- 自动匹配当前地图对应的游戏场景
- 将事件坐标从 PAL 原始坐标（16px/tile）转换为编辑器坐标（32px/tile）
- 在地图上显示事件标记（白色编号，选中时为绿色，带闪烁动画）

#### 事件编辑

- **放置事件**：切换到事件工具，在地图上点击空白位置放置新事件
- **选中事件**：点击已有事件选中，或点击事件列表中的项
- **选中居中**：点击事件列表项，地图会自动居中到该事件坐标
- **复制/粘贴**：选中事件后按 `Ctrl+C` 复制，`Ctrl+V` 在点击位置粘贴
- **删除事件**：选中事件后按 `Delete` 键删除
- **事件属性**：右侧属性面板显示事件详细属性（触发脚本、自动脚本、触发方式、图像、方向等）

#### 事件导出

- **💾 导出 JSON**：导出为 SSS-compatible JSON 格式（含全部场景或当前场景）
- **💾 导出 MKF**：浏览器内直接生成完整的 `SSS.MKF` 二进制文件
  - 导出前检测风险：如果当前场景新增了/删除了事件，会弹出警告（因为事件索引偏移会影响后续场景脚本）
  - 只修改已有事件属性（坐标、脚本等）**不会**导致索引偏移，完全安全
  - 导出 MKF 必须选择"全部场景"（单场景导出会导致文件不完整）

#### 增量更新机制

导出时遵循以下规则：
- **保留未改动的场景**：完全复制原始数据，不做任何修改
- **原地替换修改事件**：已有事件（`originalIdx` 存在）在原始位置替换属性
- **追加新增事件**：新增事件（无 `originalIdx`）追加到场景末尾
- **删除已移除事件**：原始事件被删除后，在导出中不再保留
- **重新计算索引**：所有场景的事件 ID 和 `firstEventID` 按场景顺序重新分配

#### 全局事件 ID

新增事件时，ID 从已加载的 SSS.MKF 中所有事件的最大 ID 之后开始递增，确保不会与已有事件冲突。

### 7. 属性面板

选中 Tile 后，右侧属性面板显示：
- **坐标**：当前 Tile 的 (x, y) 坐标
- **Layer0 图像/高度**：第 0 层的图块编号和高度（0-15）
- **Layer1 图像/高度**：第 1 层的图块编号和高度（0-15）
- **障碍**：是否为障碍 Tile

事件模式下显示事件属性：
- **事件 ID**：全局唯一标识
- **坐标**：事件在地图上的坐标（像素级精确）
- **触发脚本 / 自动脚本**：脚本入口编号
- **触发方式**：触发条件（如接近、按键等）
- **图像 / 方向**：事件对象的图像和朝向
- **图层**：事件所在的渲染图层

支持多选时批量应用属性（"应用属性"按钮）。

### 8. 模板系统

#### 创建模板
1. 使用 **选择工具** 选中地图上的一个区域（多选或框选）
2. 点击属性面板中的 **"📋 创建模板"** 按钮
3. 模板会自动保存到左侧「模板」标签页中

#### 使用模板
1. 切换到左侧「模板」标签页
2. 点击模板列表中的模板选中它（或 Ctrl+点击多选）
3. 切换工具栏为 **📋 模板工具**
4. 在地图上点击放置模板
5. 鼠标悬停时会有半透明预览

#### 管理模板
- **编辑**：点击模板缩略图上的「编辑」按钮，进入模板编辑器
- **删除**：Ctrl+点击选中多个模板，点击「删除选中」按钮删除
- **导入/导出**：支持将模板集导出为 JSON 文件，也可以导入之前导出的 JSON

#### 模板编辑器
- 在弹窗中编辑模板内容
- 左键绘制 · 右键删除 · 外圈可自动扩展
- 支持切换 L0/L1 图层和画笔/擦除/障碍工具
- 支持撤销/重做
- 底部显示选中图块的 L0/L1 图块 ID 和高度信息

### 9. 撤销/重做

支持 **50 步** 的撤销/重做历史栈：
- **Ctrl + Z**：撤销
- **Ctrl + Y / Ctrl + Shift + Z**：重做
- 画笔绘制、擦除、障碍、模板放置、属性应用、事件放置/删除等操作都会记录历史

### 10. 文件保存

- **💾 导出地图**：将地图数据下载为二进制文件（与原格式兼容）
- **💾 导出 GOP**：将当前图块集导出为 GOP 文件
- **💾 导出事件 JSON**：导出 SSS-compatible JSON 事件数据
- **💾 导出 SSS.MKF**：浏览器内直接生成完整二进制文件（兼容原游戏）

```
│   ├── ui.js               # 界面交互、下拉列表、缓存管理、跨图组导入、MKF 编解码
│   └── main.js             # 入口、自动加载、主循环
├── assets/                 # 标记图像资源（PNG）
│   ├── bitmap1.png         # 鼠标悬停标记（绿色）
│   ├── bitmap2.png         # 选中标记（蓝色）
│   ├── bitmap3.png         # 障碍标记（红色）

```

## 技术说明

- **解码器**：完整还原了原 C++ 代码中的 MKF 索引解析、RLE 解压、256 色调色板映射
- **SSS.MKF 编解码**：浏览器内原生实现 `demkf`/`enmkf`（MKF 偏移量表 + 子文件数据）、事件对象二进制打包（32 字节/条）、场景条目打包（8 字节/条）
- **坐标系统**：完全复刻了原 Win32 版本的等角（isometric）坐标转换算法（`tileToPixel` / `pixelToTile`）
- **文件格式**：保存的地图文件与原编辑器 100% 二进制兼容（128×128×2×2 字节）
- **渲染方式**：使用 HTML5 Canvas 2D，双层等角绘制，支持透明色（Color Key: RGB(108, 88, 100)）
- **缓存策略**：GOP 图组和地图数据按需加载并缓存，避免重复网络请求
- **图层遮挡**：支持人物对象遮挡预览，使用高度阈值重画周围 Tile 来反映遮挡关系
- **事件渲染**：事件标记使用 `bitmap4.png`，选中时带透明度 0-100% 正弦闪烁动画；事件编号白色 14px，选中时绿色

## 注意事项

1. 浏览器必须支持 ES6（所有现代浏览器都支持）
2. 由于使用 `fetch` 加载本地资源，**必须通过 HTTP 服务器访问**，不能直接用 `file://` 打开
3. 地图文件和 GOP 图组文件必须配套使用（编号匹配，如 `map0001` 对应 `gop0001`）
4. 资源文件（Pat.mkf / gop/ / map/）需要在正确的相对路径上（`../../Pat.mkf`、`../../gop/`、`../../map/`）
5. 放大后点击定位、模板预览、模板编辑器均经过修复，使用模板自带的 `tileImages` 缓存避免跨图组错乱
6. **SSS.MKF 导出风险**：如果新增/删除了事件，后续场景的事件全局索引会偏移，可能导致游戏脚本引用错误。只修改已有事件属性（坐标、脚本、触发方式等）是完全安全的

## 原项目

本项目基于 palresearch 的仙剑奇侠传研究项目中的 Win32 SDK 版地图编辑器改写。

## 开发修改记录

### 事件系统（SSS.MKF 支持）

- **SSS.MKF 浏览器内编解码**：无需 Python，浏览器原生实现 MKF 打包/解包、事件对象二进制打包、场景条目打包
- **增量更新导出**：保留未改动场景，原地替换修改事件，追加新增事件，删除已移除事件
- **坐标转换**：PAL 原始坐标（16px/tile）↔ 编辑器坐标（32px/tile），导入/导出自动转换
- **自动场景匹配**：加载 SSS.MKF 时自动匹配当前地图对应的游戏场景，也可手动选择
- **全局事件 ID**：新增事件从已有最大 ID 后递增，确保不冲突
- **导出风险检测**：导出前检测事件数量变化，警告索引偏移风险
- **事件列表交互**：点击事件列表项自动居中相机到事件坐标
- **事件标记渲染**：`bitmap4.png` 白色标记，选中时绿色 14px 编号，带透明度闪烁动画
- **事件属性面板**：支持编辑触发脚本、自动脚本、触发方式、图像、方向、图层等属性
- **自定义导出对话框**：替代系统 `confirm`，支持选择导出全部或当前场景

### 界面布局改进

- **左侧面板标签页**：图块集和模板做成切换标签页并列，点击顶部标签切换
- **右下角地图选择器**：地图下拉列表和导出按钮从左侧移到地图编辑区右下角
- **工具面板联动**：点击画笔/模板等工具显示图块集/模板标签页；点击事件工具隐藏图块集/模板，只显示事件列表
- **事件按钮布局**：事件操作按钮改为 2×2 网格排列，避免超出范围

### 核心修复与增强

- **等距坐标转换修复**：`pixelToTile` 的 `tileY` 计算改为与 C++ 一致的向零截断，修复放大后点击定位偏移问题
- **模板系统完整修复**：
  - 修复 `layer1` 图块 ID 错乱（去掉多余的 `+1` 偏移）
  - 模板编辑器、预览、缩略图统一使用模板自带的 `tileImages` 缓存，避免 GOP 切换后图块错乱
  - 添加跨图组导入机制：模板可自动在当前图组中查找匹配图块，未匹配则自动导入
- **模板高度支持**：模板数据结构扩展 `height0` / `height1`，放置时正确应用图层高度
- **多选系统**：支持 Shift+点击加选、Ctrl+框选、Ctrl+Shift+减选、Alt+减选框选
- **撤销/重做**：50 步历史栈，覆盖画笔、擦除、障碍、模板放置、属性应用、事件操作等
- **属性面板**：支持多选时批量应用图块属性，图层可见性切换时动态显示对应属性行；事件模式下显示事件属性
- **滚轮缩放**：支持 50% ~ 300% 缩放，步进 0.5 倍
- **相机拖拽**：支持鼠标拖拽平移视图，平移量按缩放比例计算

### 早期基础功能

- 添加自动资源扫描：启动时自动通过 HEAD 请求扫描所有 GOP/MAP 文件
- 添加下拉列表：GOP 选择器和 MAP 选择器，支持实时切换
- 添加缓存系统：已加载的 GOP 和 MAP 数据缓存，避免重复网络请求
- 添加默认加载：自动加载第一个可用的地图和对应图组
